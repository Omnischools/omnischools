Omnischools · Statutory filing

The GES return, generated

Every term, schools file a return with the Ghana Education Service summarising enrolment, movement, staffing, attendance, and academic performance. Omnischools pulls all of that from the system, drops it into the right form, and produces a printable PDF the headmaster signs and stamps. What used to take a Saturday afternoon now takes ninety seconds.

Updated: Attendance figures now feed from the system. The previous version of this return left the attendance section blank for hand entry; with the attendance module shipping, the section is auto-filled with class-by-class breakdowns and a school-wide rate. Special needs remains manual until that module is built.

Why this layout

  • Drawer pattern matches Record Payment, Send Reminders, and the broader transactional flow. Generation is a one-shot action; admin is selecting a few options before triggering. Modal would interrupt; drawer composes.
  • Two return types: Termly and Annual census. The "Due now" badge on Termly tells admin which one is currently expected — schools fill termly returns once per term, annual census once a year. Omnischools infers what's due from the calendar and surfaces it.
  • Period defaults to current term. The "Earlier term · pick from history" tile is for back-filling — if a school joined Omnischools mid-year and needs to file an earlier term retroactively. Most uses will be the default.
  • What's included is a checklist with status tags. Four states now: "Auto" (green, the system fills it), "Auto · NEW" (gold-bordered with NEW indicator, for sections that recently flipped from manual to system-fed — attendance is the latest), "Manual" (grey, blank space for handwriting), "Skipped" (greyed-out, not relevant for this return type). Admin sees exactly what they're getting before clicking generate.
  • Inline data preview in the meta line: "+18 admissions · −6 withdrawals · +3 transfers in · −2 transfers out" / "From 47 marked registers · 15 Sep – today · school-wide and by class". Admin can sanity-check the numbers before they show up in the PDF. If something looks wildly wrong, fix it now rather than after printing and signing.
  • "Manual" rows are warn-tinted with bold "blank fields will be left for manual entry" so admin knows they'll need to fill those in by hand. Honest about what the system can and can't auto-fill — better than silently producing a return with missing sections that gets rejected by GES. The list of manual rows shrinks as new modules ship; this version of the return has only one manual section (special needs) where the previous version had two.
  • "NEW" indicator on the attendance row calls out what's changed since the previous return. Admin who filed last term by handwriting attendance values will notice immediately that this term that section is auto-filled. Builds trust — "the system is honestly telling me what's new."
  • Skipped row for "Infrastructure indicators" explains why it's skipped — "Not required on termly returns · only on annual census." Educates admin about the GES forms structure without making them look it up.
  • Output card shows file format, page count, and filename ahead of generation. The filename uses a sensible convention (`CTK_GES_T1_2025-26.pdf`) admin can recognise in their downloads folder. The italic line beneath gives the next steps after download — a small closing-the-loop touch.
  • Footer summary "6 sections auto-filled · 1 left blank" compresses the checklist into one digestible fact. Admin sees the headline number; the detailed checklist is there if they want it. The number ratchets up as more modules feed the return.

Why this layout

  • Six A4 pages, portrait orientation. A4 because Ghana standard is metric paper, not US Letter. Portrait because GES forms have always been portrait — the District Officer flipping through a pile of returns expects portrait. Going landscape would feel wrong even if it fit data better.
  • Cover page is restrained, not promotional. "Ghana Education Service" tag at the top establishes the document's authority before the school name. The school name in Fraunces with "JHS" italic gold preserves the Omnischools brand language without making the cover feel like marketing. Three-stat filing summary gives the District Officer the headline numbers before they turn the page — useful when they're processing dozens of returns.
  • Navy header band on every content page with school crest + name + return type + GES School ID. Dense identification on every page so a District Officer sorting loose pages back together can always tell which school a page belongs to. The crest "CK" in gold-on-navy reuses the Omnischools logo treatment — consistent with the receipt design.
  • Section numbering throughout (1–10). Every section has a Fraunces italic gold numeral and an uppercase tracked-out title. This mirrors the GES form structure where sections are numbered and referenced. A District Officer can say "Section 4 is incomplete" and admin knows exactly where to look.
  • "Source" tag on every section head — "from school settings", "from staff records", "from attendance module", "manual entry", "as of 19 Dec 2025". Auditor knows where each section's data originated. The source tag is itself an audit signal — when "manual entry" flips to a system source on a future return, the District Officer can see at a glance that the school upgraded its tooling.
  • Identification grid in 2×5 — exactly the GES form pattern. Bordered cells with all-caps tiny labels and Fraunces values. The GES School ID rendered in mono signals it's a system identifier, not free text.
  • Enrolment table includes "Last term" and "Δ" columns. GES forms historically have a "previous period" column for trend tracking. Showing the delta in green/terra/em-dash mirrors the school stats report — visual consistency. The em-dash for unchanged values (instead of "0") matches our convention from elsewhere in the system.
  • Movement section uses hybrid card+table layout. The four movement cards (admissions, withdrawals, transfers in, transfers out) at the top with green/terra left borders give the District Officer a read-at-a-glance summary. The detailed by-class table beneath provides the breakdown for verification. Withdrawals broken down by reason (Relocation / Financial / Other) — qualitative data that contextualises the number.
  • Auto-fill block for attendance has a green tick icon and "Auto" pill — the visual opposite of the manual-fill block (which has a warn icon and hatched background). Same shape and field structure, different colour vocabulary. The District Officer scanning multiple schools' returns can tell at a glance which schools have system-fed attendance vs hand-filled; both are valid, both are legible, but they're visually distinct.
  • Four auto-fields in the attendance block: boys avg, girls avg, total student-days, term rate. All four are computed from the marked registers; the system shows its work. "↑ 2.1% vs Term 3 2024/25" delta in the rate field uses the same green/terra delta convention as the enrolment table. Visual consistency: deltas mean the same thing wherever they appear.
  • Provenance line under the attendance block in italic small caps: "14,664 student-days enrolled · 13,706 attended · last register marked Fri 21 Nov 8:14 AM by Mr. Mensah · headteacher attests by signing the declaration." Five facts that ground the numbers in real events: total opportunity, total realised, when the data stops, who marked the last register, and where the accountability sits (the headteacher's signature on page 6).
  • By-class attendance table below the auto-fill block shows the same data broken down 9 ways (one row per class) plus a total row. Green dots on the "Total avg" cells indicate auto-fill at the cell level; admin scanning the table sees which numbers came from the system. The JHS 3C row stands out with its 70.3% rate — the explanatory callout below acknowledges the anomaly without burying it.
  • JHS 3C anomaly callout in gold-bordered italic: "70.3% rate reflects an unusually high absence pattern this term. The class teacher reassignment in week 6 (Mr. Kwakye to Ms. Owusu) coincides with the dip." The system surfaces the anomaly and contextualises it rather than letting it sit unexplained. The District Officer reading this knows the school knows; not silently hoping no one notices the outlier.
  • Manual-fill blocks remain visually distinct for sections genuinely not yet captured (special needs). Diagonal-line hatched background, dashed border, warn-tinted icon. The visual difference between auto-fill (clean, ticked, green) and manual-fill (hatched, alert, warn) tells the story of which modules are built — and which aren't yet. As more modules ship, the hatched zones shrink and the clean auto-fill zones grow.
  • Pupil-teacher ratio callout ("22.3 to 1 — within recommended range for JHS") gives interpretive context to a number that's a key indicator for GES. Saves the District Officer from doing the calculation; saves admin from being asked about it.
  • Performance section uses the BECE 1–9 scale with explicit reminder callout: "1 is the highest and 9 is the lowest." Same convention from the school stats academic performance page. Best/worst class highlights in green/terra at the top so the eye lands on extremes first. Without the scale reminder, anyone unfamiliar with BECE conventions could misread the data.
  • Signature row is paired: Headteacher on the left (filled with simulated handwritten signature, terra-bordered school stamp at -8° rotation), District Education Officer on the right (blank, with "To be filled at submission" italic placeholder). The blank right block is a deliberate signal that the District Officer is the next step in this document's life.
  • Stamp aesthetic matches the receipt PDF — terra circular border, multi-line content (school name, level, location + established year), faint background tint to suggest ink on paper. Slight rotation makes it feel real; perfectly horizontal would look fake.
  • "To be filled at submission" on the District Officer's signature block is a small humility — the document is not complete on generation, only on filing. The system isn't claiming finality; it's producing a correctly-prepared form that the District Officer signs to accept.
  • Filing instructions updated to reflect attendance is now auto-filled. Old step "Complete the manual-entry sections (special needs, attendance) using the school's physical register" became two steps: complete the special-needs section, and verify the auto-filled attendance figures before signing. The headteacher's signature attests to ALL reported numbers — including the auto-filled ones. Auto-fill doesn't reduce accountability; it just reduces typing.
  • Footer band on every page: school name + GES ID centre, "Generated by Omnischools" right. Same "powered by" treatment as the receipt PDF — gold-italic Omnischools wordmark, restrained. The school's identity dominates; Omnischools is acknowledged but doesn't compete.
  • Page numbering style: "Page 2 of 6" at the bottom, Fraunces italic. Not strict navigation; just confirms the document is whole. If a page goes missing in transit, the District Officer notices.
