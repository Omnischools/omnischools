Taking a view out.

GES work does not stay inside a dashboard — figures go into ministerial briefs, regional reports, budget submissions, parliamentary answers. The exports surface is where any Oversight view becomes a CSV or a PDF a director can carry into that work. But an export is the moment data leaves the platform's controls, so this surface carries forward two disciplines the rest of Oversight established. The coverage caveat travels with the data — an exported file states, on its face, that it reflects a known share of schools, so a figure can never be quoted downstream as if complete. And exporting named records goes through the very same audit logging as viewing them on screen — leaving the platform is not a way around the gate. The demo is a district director exporting the Wassa Amenfi West enrolment analysis. This is the final surface of the Oversight product line.

An aggregate export is not audit-logged · it carries no named records

An export carries the platform's discipline with it

  • The coverage stamp cannot be switched off. Every other toggle is optional; the coverage stamp is fixed on. An exported file states on its face that it reflects 34 of 40 schools — because the moment data leaves the platform, the surrounding context that kept it honest is gone, so the caveat has to be inside the file itself.
  • The stamp is specific, not a disclaimer. It names the actual numbers — 34 of 40, 85%, this district — and says plainly the figure is not a complete count. A vague "data may be incomplete" footer would be ignored; a specific stamp is something a reader in a ministry brief has to reckon with.
  • Method and provenance can travel too. The stage-for-stage method note, the GSS census vintage, the WAEC extract date — optional toggles, on by default. An enrolment figure quoted without its method invites misreading; the export can carry its own explanation.
  • The summary names whether the export contains named records. "None — aggregate" here, in green. Before a single file is generated, the officer sees whether they are exporting an aggregate view or named data — because those two cases are governed completely differently, as section 2 shows.
  • An aggregate export is not audit-logged — and the surface says so. Aggregate data leaving the platform is ordinary GES reporting; it carries no named records and needs no log entry. The surface is precise about the line: the logging discipline attaches to named records specifically, not to exporting in general.
  • Exports stay within jurisdiction. A district director exports district views — the same ceiling as every other surface. The export path is not a gap in the scope model; it inherits it.

Exporting a named record is the same action as viewing one
If an export includes any named student or teacher record, it goes through the identical justification gate as the compliance record view, and it writes the same entry to the Oversight access & audit log. Putting a name into a CSV is not a way around the gate — the file leaving the platform is exactly what the gate exists to control. Aggregate exports are unaffected; this applies only when a name is in the file.

Leaving the platform is not leaving the rules

  • The export path is not a loophole. The obvious gap in any oversight tool is "view is controlled, but export is not" — so a determined user pulls a CSV instead. This surface closes that: a named-record export hits the identical gate and writes the identical log entry as viewing the record. The file leaving is exactly the event the gate governs.
  • The export history names which exports were logged. Three aggregate exports marked "not logged," one named-record export marked "Logged · R-0094." The history itself shows the rule operating — the officer and any reviewer can see at a glance which exports carried names and were therefore logged.
  • The on-screen access and the export are the same event. R-0094 in the compliance record view, R-0094 in the audit log, R-0094 here as a PDF export — one logged event, not three. The platform records the access to the named record; the format it left in is just an attribute of that one event.
  • Aggregate and named-record exports are treated differently, and the line is clear. Aggregate data is coverage-stamped but not logged — ordinary reporting. Named-record data is gated and logged. The surface never blurs the two; the discipline is proportionate to what is actually in the file.
