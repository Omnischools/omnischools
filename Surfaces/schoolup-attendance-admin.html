Why this layout

  • Five vertical regions, top to bottom: Pulse → By class → Trend → Attention → Edit requests. Most-actionable at the top, most-reflective at the bottom. Headmistress doing a 30-second morning glance gets what she needs in regions 1-2; if she has more time, regions 3-5 are there.
  • Region 1 — Pulse has a hero tile (1.4× wide) with the school-wide attendance number, plus four supporting tiles. The hero uses navy bg + gold accent — same brand language as the Cover page on the GES return. The hero attendance number is in gold, not because it's good or bad, but because it's the headline.
  • Hero tile breakdown bar shows the four-status proportions inline. 281 / 6 / 4 / 21 across the bar, with a legend below in mono. Same colour mapping (green present / gold late / warn excused / terra absent) used everywhere in attendance.
  • "Registers marked: 7 of 9 · 2 pending · JHS 1B, JHS 3C" calls out the names of unmarked classes inline. Headmistress doesn't have to scroll to Region 2 to know which teachers haven't marked. Specificity over generality.
  • "vs yesterday: +2.1%" is the smallest tile but most contextual. A 93.2% number alone is meaningless; "we improved 2.1% from yesterday" gives the headmistress directional sense. Yesterday's exact number ("Yesterday 91.1% · 285 of 313") in the sub-line for transparency.
  • Region 2 — By class as a 3×3 grid matching the school's actual class structure (JHS 1A/B/C, 2A/B/C, 3A/B/C). The grid mirrors physical school structure — the headmistress can scan the cards in the same mental order she walks the corridors.
  • Each class card has a horizontal stacked bar showing today's attendance proportions with embedded counts. "32 present · 1 late · 2 absent" rendered as colour segments with numbers inside. Card with high present count looks mostly green; card with mostly absent looks mostly terra. Visual scan = problem detection.
  • Pending classes get terra-tinted background and pulsing pill. JHS 1B (Mr. Asante hasn't marked) and JHS 3C (Mr. Kwakye out, no cover assigned) stand out from the green-tile sea. The pulsing animation is subtle but draws attention without being aggressive.
  • Excursion class shows different language: "33 on trip" instead of "33 present", "back 3:30 PM" instead of timestamp, green excursion pill. The data model knows it's an excursion; the card reflects it.
  • JHS 3C · "covering teacher needed" footer line surfaces an action item without an explicit button. Headmistress reads "Mr. Kwakye out today · no cover assigned" and knows what she needs to do. The dashboard names problems; the headmistress solves them.
  • Region 3 — Term trend is the only chart on the page and it's earned its space. Single metric (school-wide attendance) over time, with reference lines, holiday bands shaded, and today's point highlighted in gold. The chart uses the same green as the Present colour — visual continuity.
  • Trend headline shows three numbers: term average (93.8%), comparison to previous term (91.4%), and delta (↑ +2.4%). All three exist because each answers a different question. "Are we doing well?" needs the average. "Are we improving?" needs the delta. "Vs what?" needs the previous term."
  • Trend footer with five mini-stats: Best day, Lowest day (with annotation: "Fri 17 Apr · rain"), Mondays avg, Fridays avg, vs last term. Day-of-week breakdown reveals patterns — the school's Friday average being meaningfully lower is real and worth knowing.
  • Toggle group: School / By class / By gender. The default view is school-wide; admin can pivot to compare classes (line per class) or gender (two lines, M and F). Default is the "boring" view; the slices are there when needed.
  • Region 4 — Students needing attention is the dashboard's destination. Tabs: All flagged / Below 70% / Long absence / Pattern shift. "All flagged" tab is in terra colour because it's the catch-all for everything; specific tabs are gold-styled.
  • Each row has flag, percentage, sparkline, action. Flag column shows what kind of attention (Long absence / Pattern · Fridays / Below 70% / Pattern shift) with detail line ("3 consecutive days · no contact from home"). Percentage with denominator. Sparkline showing 14-day attendance pattern. Action button.
  • Sparkline uses real attendance status colours — green present, gold late, warn excused, terra absent. The same colour vocabulary throughout the system so headmistress doesn't have to re-learn legends. Weekend bars are short stubs; holidays would be hatched (none in this 14-day window).
  • Action button changes by flag type: "Follow up →" (gold primary) for terra/warn flags that need contact; "Check in" (subtle) for excused absences where no urgent action is needed but headmistress should know. Visual weight matches urgency.
  • Stephen Opoku's flag detail: "BECE candidate" adds context that turns a generic "below 70%" into a specific worry. JHS 3 students sitting BECE need attendance for the school's pass-rate stats and for the student's own preparation. The dashboard knows; the flag says so.
  • Region 5 — Edit requests is small most days. Two pending here: Mr. Mensah wants to fix Kwame's Late status, Ms. Adjei wants to change Yaa from Absent to Excused with a doctor's note. Both have legitimate reasons; both need approval.
  • Each request shows: who, which register, when submitted, when requested (delta from edit window), what change, why. "Submitted 30 Apr · 8:11 AM · request made 2 May · 9:18 AM · 52 hours after edit window" gives the headmistress audit context — was the request prompt or stale?
  • Reason quoted in the gold-bordered callout: "Change requested: Mark Kwame Adu as Late instead of Present. He arrived at 8:31 AM but I marked Present in error before he came in." Direct teacher voice — headmistress reads what they actually said, not a summarised version.
  • Approve / Reject buttons inline with each request. Headmistress doesn't have to click through to a separate approval screen for routine requests. The dashboard handles the common case; complex requests can drill through to the audit log via a "more detail" link (not shown here, but the design assumes it).
  • Approve = green, Reject = white-with-terra-border. Visual asymmetry — approving is the easy default, rejecting requires conscious choice. Both single-click but visually different weights.
